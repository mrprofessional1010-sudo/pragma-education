1 Project Overview

What Does This Website Do?

Pragma Education is an educational coaching website that:

  • Showcases courses for competitive exams (IB ACIO, UPSC)
  • Provides information about the institution and faculty
  • Allows students to register and login
  • Offers study resources and current affairs updates
  • Displays success stories and testimonials
In Simple Terms: It's like an online brochure + portal for a coaching institute.

Type of Website: Static Website

Term Meaning
Static Pages are pre-written HTML files. Content doesn't change based on user.
Frontend Everything runs in the browser (client-side).
No Backend No server processing. No database. Just files served directly.

Why This Approach?

  1. Simple to host - Just upload files to any web server
  2. Fast loading - No server processing needed
  3. Easy to maintain - Edit HTML files directly
  4. Cost-effective - Can be hosted for free on GitHub Pages, Netlify, etc.
  5. Perfect for information websites - Doesn't need user accounts or dynamic content

6 Animations & Interactions

AOS (Animate On Scroll)

<div data-aos="fade-up">
    This will fade up when scrolled to
</div>

<div data-aos="fade-up" data-aos-delay="200">
    This will fade up after 200ms delay
</div>

Available Animations

Animation Effect
fade-up Fades in from bottom
fade-down Fades in from top
fade-left Fades in from right
fade-right Fades in from left
zoom-in Zooms in
zoom-out Zooms out

Customization Options

data-aos-duration="1000"   <!-- Animation duration (ms) -->
data-aos-delay="500"       <!-- Delay before starting -->
data-aos-once="true"       <!-- Only animate once -->
How to Disable an Animation:
Simply remove the data-aos="..." attribute from the element.

7 State & Logic

No Complex State Management

This is a static website - it doesn't store user data or have login sessions.

The forms (login, register, contact) currently just show alerts:

document.getElementById('loginForm').addEventListener('submit', function(e) {
    e.preventDefault();  // Stop form from submitting
    alert('Login functionality will be connected to backend.');
});
To make forms work for real: You'd need to add a backend (Node.js, PHP, etc.)
